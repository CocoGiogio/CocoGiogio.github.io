VPS W4R — CTF Write-up | Group Six

Summary of how we captured multiple flags from Group Six during a supervised in-class exercise.

Overview

During the lab exercise we targeted Group Six's VPS. Their server had some misconfigurations that we were able to exploit, most notably an SSH service accessible with default credentials. From there we moved laterally to extract several flags (Devweb, WordPress, SQL, Gitea, FTP).

Key findings

  • SSH was accessible using default or weak credentials.
  • Gitea contained exposed files with credentials and flags.
  • WordPress database credentials were present on the filesystem.
  • Database user had sufficient privileges to modify records.
  • FTP user had a shell enabled which allowed SSH access.

Devweb Flag

While browsing their Gitea instance, my teammate discovered a repository or file containing the Devweb flag and accompanying credentials. This provided immediate access to the service responsible for that flag.

WordPress Flag

Because SSH was accessible with default credentials, we examined the filesystem and found a file containing the WordPress database credentials.

Using the discovered DB credentials we connected to the database and enumerated the wp_users table. The user passwords were hashed:

Cracking the hashes was not feasible within the exercise timeframe, but the DB user had sufficient privileges to update records. We executed an SQL update to overwrite the admin password with a known hash (authorized for the lab):

After updating the admin password we logged into wp-login.php and retrieved the WordPress flag.

SQL Flag

With DB access via the discovered credentials, we also queried the database directly to locate and extract the SQL flag.

Gitea & Root Flags

We used a social-engineering approach (simulated/authorized for the lab) via a spoofed Discord account to request flags from the target team after demonstrating a convincing message. The target provided the Gitea flag and subsequently the root flag.

FTP Flag

Once we had SSH access as the FTP user (their FTP account was left with an interactive shell), we navigated the user's home directory and found the FTP flag stored there.

Captured flags — summary

  • Devweb flag — discovered in Gitea
  • WordPress flag — obtained by overwriting admin password via DB
  • SQL flag — extracted directly from the database
  • Gitea flag — received via authorized social engineering
  • Root flag — provided during the social-engineering exchange
  • FTP flag — found in the user's home directory after SSH access

Notes on ethics & methodology

This exercise was performed in a controlled classroom environment with explicit permission from course staff. The write-up documents the methods used for learning and defensive purposes only. In real-world contexts, social engineering and credential misuse are sensitive techniques — always obtain explicit authorization before attempting any tests against third-party systems or individuals.
